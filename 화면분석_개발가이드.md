## 0. 한눈에 보기

샘플은 **단일 HTML 파일**(프레임워크·백엔드 없음, 인라인 CSS + 바닐라 JS)이고, 모든 콘텐츠가 JS 객체(`TX`, `COUNTRY_MEDIA`)에 **하드코딩**돼 있는 정적 목업이다. 화면 골격·디자인은 완성도가 높으므로, 개발의 본질은 **"하드코딩 데이터 → 수집 파이프라인이 생성하는 JSON으로 교체"** 하는 것이다.

공통 패턴 3가지 (전 화면 재사용):

- **카드 → 모달**: 카드를 클릭하면 `showModal()`이 상세(detail) + 인사이트(액션플랜) + 출처(sources)를 띄움. 글로벌동향/자회사/TopicWatch가 동일 구조.
- **카테고리 태그**: 시장·경제 / 금융산업 / 규제·리스크 / 전략·운영 / ESG → 색상 클래스(`t-econ`, `t-fin`, `t-risk`, `t-digital`, `t-esg`)로 통일.
- **이중언어(KR/EN)**: `setLang()`로 전체 재렌더. 단 현재 EN 데이터는 stub(한글 대비 대부분 1개 항목만 채워짐).

## 1. 화면별 항목·콘텐츠 분석 (참여코너 제외)

### 1.1 글로벌 동향 (Brief) — 메인 화면

화면에서 **위 → 아래** 순서로 6개 블록:

| # | 블록 | 보여주는 내용 | 데이터 키 |
|---|---|---|---|
| 1 | 주간 초점(Hero) | 큰 헤드라인 + 설명문 + 키워드 칩, 오른쪽에 "KB 액션 ①②③④" 박스 | `heroH`, `heroP`, `heroChips`, `focusP` |
| 2 | 빅넘버 2장 | 핵심 지표 2개 (예: 인니 BI-Rate 5.75%, Fed 3.50–3.75%) + 맥락 설명 | `bnEye`, `bnDesc`, `bnCtx` |
| 3 | 4대 시장신호 | 카테고리 태그 + 헤드라인 + 한줄 요약 카드 4장. **클릭 → 모달**(상세분석 + 💡액션플랜 + 출처) | `signals[]` |
| 4 | 글로벌 분석 요약 | 트렌드 2~N개 (제목 + 설명) | `trends[]` |
| 5 | 주간 현장(KPI) | 지표 셀 (예: 美 CRE 연체율 1.2%) | `kpis[]` |
| 6 | 거점 현장의 한마디 | 국가 플래그 + 거점 + 시각 + 인용문 (가로 스크롤) | `reporters[]` |

`signals[]` 한 건의 구조 (모달까지 포함, 가장 복잡한 단위):

```js
{ n:'01', cat:'t-econ', label:'시장·경제',
  h:'헤드라인', p:'한줄요약',
  detail:'<p>상세 분석 본문…</p>',
  insight:'<p><strong>[본점 Action]</strong> 액션플랜…</p>',
  sources:[{title:'원본 리포트명', url:'#'}] }
```

> 성격: **편집·합성(editorial)** 화면. 개별 기사가 아니라 한 주를 요약·해석한 결과물이라 자동화 난도가 가장 높다.

### 1.2 현지언론 Intelligence (Countries) — 파이프라인 직결 화면

| 블록 | 보여주는 내용 |
|---|---|
| 인트로 | "사내 리포트가 아니라 현지 주요언론 기사를 선별" 운영 원칙 명시 |
| 필터 버튼 | 전체 / 금융(finance) / 디지털(digital) / ESG / 금융사고·리스크(risk) |
| 기사 피드 | 기사 카드: 국가 플래그+이름, 카테고리 배지, **매체·날짜**, 제목, 요약, **KB 시사점**, **원문 링크** |
| 커버리지 그리드 | 국가별 소스 현황 카드. 기사 있으면 `ACTIVE`, 없으면 `SOURCE WATCH` + 분석 건수 |

`COUNTRY_MEDIA` 구조 (이게 **수집기 산출물과 거의 1:1**):

```js
{ flag:'🇸🇬', n:{ko:'싱가포르',en:'Singapore'},
  s:'The Business Times · The Straits Times',   // 소스 매체 목록
  a:[ { c:'esg risk',                            // 카테고리(복수 키워드 가능)
        b:{ko:'ESG · 리스크',en:'ESG · RISK'},   // 배지 라벨
        src:'The Business Times', d:'2026.06.12', // 매체·날짜  ← 수집기
        t:{ko:'제목',en:'…'},                     // 제목       ← 수집기
        q:{ko:'요약',en:'…'},                     // 요약       ← AI
        k:{ko:'KB 시사점',en:'…'},                // 시사점     ← AI
        u:'https://…' } ] }                       // 원문 URL   ← 수집기
```

현재 데이터 상태(샘플): 10개 시장 중 **싱가포르 1 · 중국 2 · 인도 1 = 총 4건만** 실기사이고 나머지는 빈 배열(`a:[]`) → SOURCE WATCH.

> 국가 목록 주의: 샘플의 10개 시장은 GB·US·JP·HK·SG·CN·VN·IN·MM + **오클랜드(NZ)**. 프로젝트 관리국 11개(CLAUDE.md) 대비 **ID·KH는 자회사 화면으로 분리(정상)**, **NZ는 프로젝트에 없는 추가 국가**다. NZ 포함 여부를 결정해야 한다.

### 1.3 국외연결자회사 (Subsidiaries)

| 블록 | 보여주는 내용 |
|---|---|
| 카드 그리드 | 자회사별 카드(제목+요약). **클릭 → 모달**(경영실적 상세 + 액션플랜 + 출처) |

현재 2개: **PT Bank KB Indonesia Tbk**(구 부코핀, 흑자전환), **KB 프라삭 은행**(캄보디아, 상업은행 전환). 구조는 `signals`와 동일(`{h, p, detail, insight, sources}`)이라 모달 로직 공유.

> 성격: 뉴스보다 **경영공시·IR 기반**, 분기/반기 cadence. 자동화 비중 낮고 수동 큐레이션 위주.

### 1.4 Topic Watch

| 블록 | 보여주는 내용 |
|---|---|
| 카드 그리드 | 주제 카드(제목+요약). **클릭 → 모달**(리포트 상세 + 액션플랜 + 출처) |

현재 1개: **[ESG] 유럽 新 환경 규제**. 구조 `{h, p, detail, insight, sources}`.

> 성격: 여러 국가에 걸친 **횡단 이슈**(ESG·규제·디지털)를 주제로 묶은 deep-dive. 시장신호의 심화판.

### 1.5 규제·금융기관 (Regulations)

현재 **placeholder만** ("규제 정보·유관기관 협력 소식 업데이트 예정"). 실제 콘텐츠 없음.

> 개발 시 Topic Watch와 동일한 카드+모달 구조를 그대로 쓰고, **규제 전용 소스**(BCBS·EU·각국 감독당국/중앙은행)와 규제 카테고리만 추가하면 된다.

## 2. 기존 파이프라인과의 매핑

`glb-one-teams`는 "AI 없이 수집·필터·중복제거"만 담당(88 sources / 106 feeds). 화면이 요구하는 필드와 대조하면 **수집기가 주는 것**과 **새로 만들어야 하는 것**이 갈린다.

| 화면 필드 | 출처 | 상태 |
|---|---|---|
| 매체(src) · 날짜(d) · 제목(t) · 원문 URL(u) · 국가 | 수집기 | ✅ 이미 있음 |
| 카테고리(c) — finance/digital/esg/risk | 분류 AI | ⬜ 신규 |
| 요약(q) | 요약 AI | ⬜ 신규 |
| KB 시사점(k) | 시사점 생성 AI | ⬜ 신규 |
| 유의미 기사 선별 (ACTIVE vs SOURCE WATCH) | 스코어링 규칙+AI | ⬜ 신규 |
| 주간 focus / 4대 신호 / Topic 리포트 / 인사이트 | 편집 합성 AI + 사람 검수 | ⬜ 신규 |

### 권장 아키텍처

```
[수집기 glb-one-teams]                         [신규 AI 레이어]                    [프론트]
 fetch → filter → dedup ──→ DB(SQLite) ──→ classify → summarize → KB시사점 ──→ data/*.json ──→ GitHub Pages
 (AI 없음, 현행 유지)                         + select(스코어링)                  (fetch로 로드)
                                             + synthesize(brief/topic, 사람검수)
```

핵심 원칙: 수집기는 **현행대로 AI 없이** 두고, **AI를 별도 모듈/단계로 추가**한다 (프로젝트 "다음 과제: AI 프로바이더 실험"과 정확히 일치). 서버 IP가 Google News RSS 503 차단이므로, 기존 `맥북 수집 → rsync` 패턴을 **맥북 수집 → JSON 생성 → Pages 레포 push**로 확장한다.

### 데이터 계약 (화면 ↔ 데이터)

화면별로 JSON 1개를 `data/` 아래 둔다. `COUNTRY_MEDIA` 형태를 그대로 표준 스키마로 삼으면 된다.

```
data/
 ├─ brief.json        # hero, bignum, signals[], trends[], kpis[], reporters[]
 ├─ countries.json    # COUNTRY_MEDIA[] (국가 × 기사[])
 ├─ subsidiaries.json # subs[]
 ├─ topics.json       # topics[]
 └─ regulations.json  # topics[]와 동일 스키마
```

프론트는 하드코딩된 `TX`/`COUNTRY_MEDIA`를 지우고 `fetch('data/countries.json')`로 교체. 렌더 함수(`renderCountries` 등)는 거의 그대로 재사용 가능.

## 3. 개발 순서 (로드맵)

**Phase 1 — 현지언론 Intelligence 엔드투엔드 (먼저)**
수집기 산출물 중 매체·날짜·제목·URL을 `countries.json`으로 export → 프론트를 fetch 기반으로 전환. **AI 없이도** 카드가 뜨는 데이터 흐름을 먼저 증명(요약·시사점은 비워두고). 가장 쉽고 파이프라인 검증에 최적.

**Phase 2 — AI 분류·요약·시사점 모듈**
기사 단위로 카테고리(c) 분류 + 요약(q) + KB 시사점(k) 생성. 여기서 프로바이더 실험. ACTIVE/SOURCE WATCH 선별 스코어링도 이 단계.

**Phase 3 — Topic Watch + 규제·금융기관**
기사들을 주제로 클러스터링 → 주제 리포트 생성. 두 화면이 같은 카드+모달 구조라 한 번에. 규제는 전용 소스만 추가.

**Phase 4 — 글로벌 동향(Brief)**
Phase 2~3 결과를 **편집·합성**해 주간 focus + 4대 신호 + 트렌드/KPI 생성. AI 초안 → **사람 검수** 권장(KB 액션플랜은 책임 있는 메시지라 자동 발행 비권장). 인용문·KPI 일부는 수동 입력 여지.

**Phase 5 — 자회사**
IR·경영공시 소스 추가, 분기 cadence로 별도 운영. 수동 큐레이션 + AI 요약.

**상시(공통)**: ① EN 데이터 채우기(미정이면 KR 전용으로 축소 결정) ② 하드코딩 날짜(`TODAY='2026-06-23'`) 동적화 ③ JSON 생성·push 자동화(정기 수집 자동화 과제와 연결).

## 4. 기술 메모 · 주의점

- **모달 로직은 공유 자산**: `showModal(type, idx)`가 현재 brief/country/sub/topic을 처리(규제는 placeholder라 미연결 — 동일 패턴 그대로 재사용 가능). 새 화면도 `{h, p, detail, insight, sources}` 스키마만 맞추면 끝.
- **카테고리 키는 부분일치**: `a.c`가 `'esg risk'`처럼 복수 키워드 문자열이고 필터가 `includes()`로 매칭 → 한 기사에 다중 태그 가능. 분류 AI 출력도 이 규칙에 맞춰 키워드 배열/문자열로.
- **출처 링크**: 신호·자회사·토픽의 `url`은 현재 전부 `'#'`(죽은 링크). 실제 수집 URL로 채워야 함. 현지언론 기사만 실 URL 보유.
- **국가 정합성**: 수집기 `sources.yaml`의 국가코드 ↔ 화면 `COUNTRY_MEDIA` 플래그/국가를 단일 소스로 동기화. NZ 추가 여부 먼저 확정.
- **이중언어 비용**: EN까지 가면 요약·시사점을 2배 생성(또는 export 시 AI 번역). 초기엔 KR 우선 권장.
- **레포 분리**: 샘플은 `uandix-kaneiko` 계정 Pages, 프로젝트는 `lSusial/glb-one-teams`. 화면 레포와 데이터 생성 레포의 연결(서브모듈/CI/수동 push) 방식을 정해야 함.
```
